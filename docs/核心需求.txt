1.在 dev 环境下需要导入测试数据，所以可以将正式服的数据导出来，然后导入到 dev 服。但是会遇到一些问题，
导出数据时，标签和账单分类会导出，但是导入的时候，如果没有预设的标签、账单分类和账户，就会导入不了。考虑是否
增加功能可以在导入的时候，自动创建没有的账户、标签和账单分类。

2.数据的兼容问题，现在的交易记录是普通的交易，如果要支持投资资产，是否得考虑数据兼容性，以当前持有的基金为例，
需要添加基金名称、基金代码、基金净值等信息。怎么考虑普通交易记录和投资资产记录数据的兼容性，万一以后还有其他
类型的投资资产，又需要怎么兼容？

3.理财和记账需不需要拆分，记账偏向记录生活记账，理财偏向记录投资资产。考虑在当前界面的左上角加一个切换功能，点击
则出现一个动画效果切换到理财界面，或者切换到记账界面。

4.例如基金，把每只持有过、自选基金的每日信息记录到数据库里，加强理财数据的查询能力，防止免费 api 失效之后无法
查询数据的问题，这个需求是否合理，假如以后假如资产，例如股票，怎么考虑兼容性？

5.理财模块需要的功能，可以追踪日、周、月、年的收益，对资产的表现进行分析，可以产生月报，可以预设策略，根据这个
策略系统可以提醒用户是否应该操作，可以进行资产分析，资产的分布等等，可以参考各大理财软件。

6.还有一些我没想到的，帮我补充。
